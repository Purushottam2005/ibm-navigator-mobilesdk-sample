  • IBMECMRepository

    Represents a repository managed in IBM Content Navigator.

    Each repository object is an instance of a given repository type defined in IBMECMRepositoryType. A repository object is globally defined and then associated with each IBM Content Navigator desktop. Each desktop usually contains a default repository. A repository is usually where authentication occurs if container managed authentication is not performed. You can login into the main application and then authenticate with additional repositories defined within the desktop.

    As of now, P8 and CM8 repository types are supported

  • IBMECMDesktop

    Represents an IBM Content Navigator desktop that the current application is accessing. It is usually the second level object after the IBMECMApplication object.

    An application can have many desktops and only one default desktop. Typically an application accesses one desktop at a time.

    A desktop object manages lists of favorites, repositories, synced items, and other items. It also contains information about server connections and user information. For each desktop, it will also have one default repository.

    A desktop must be loaded before it can be used by calling IBMECMApplication.loadDesktop(IBMECMCompletionHandler)

  • IBMECMApplication

    Represents the main application or device interfacing with this SDK.

    An application object is the top-level object of this SDK. It holds all the IBMECMDesktop objects that this application will be accessing. A default desktop is usually set in the IBM Content Navigator server and is provided to this SDK by the baseURL set in IBMECMFactory.sharedInstance.getApplication(String). You can also provide or switch to a different desktop by providing a desktopId param in the baseURL.

    Typically you will want to call loadDesktop() to load each desktop information. By loading the desktop, you will bring down all the settings, information, and repositories this desktop can access. You can also call login() to authenticate to the default repository for the default desktop.

    There should only be one application instance that represents your application. You can obtain this singleton instance from IBMECMFactory.sharedInstance.getApplication(String)
